GLPI — Guide utilisateur

Guide synthétique pour comprendre et utiliser GLPI dans un contexte de support, d'inventaire et de gestion du parc informatique.

HelpdeskInventaireCMDB

Objectif

GLPI aide une équipe informatique à centraliser les demandes des utilisateurs, suivre le parc, documenter les actifs et garder l'historique des interventions. L'objectif quotidien est simple : savoir qui demande quoi, sur quel équipement, avec quelle priorité, et quelles actions ont été menées.

Premiers pas

  1. Se connecter à l'interface web GLPI avec le compte fourni par l'administrateur.
  2. Vérifier son profil, son entité et le périmètre visible.
  3. Repérer les menus principaux : assistance, parc, gestion, outils, administration et configuration.
  4. Utiliser la recherche pour retrouver un ticket, un utilisateur, un ordinateur ou un document.
Les menus visibles dépendent du profil. Un utilisateur final, un technicien et un administrateur ne voient pas nécessairement les mêmes fonctions.

Créer et suivre un ticket

Création

Un ticket décrit une demande, un incident ou une intervention. Il doit contenir un titre clair, une description exploitable, le demandeur, la catégorie et, si possible, l'actif concerné.

Traitement

Le ticket peut être attribué à un technicien ou à un groupe. Les suivis, tâches, documents joints et changements de statut constituent l'historique de traitement.

Clôture

Une fois la solution apportée, le ticket est résolu puis clôturé. La solution doit rester compréhensible pour faciliter les recherches futures et la capitalisation.

Consulter le parc

Le parc GLPI regroupe les actifs de l'infrastructure : ordinateurs, moniteurs, logiciels, imprimantes, équipements réseau, périphériques, téléphones, racks et autres objets techniques. Selon la configuration, ces actifs peuvent être saisis manuellement ou alimentés par inventaire automatique.

  1. Ouvrir le module de parc ou d'actifs.
  2. Choisir le type d'élément recherché.
  3. Filtrer par nom, numéro de série, utilisateur, lieu, statut ou entité.
  4. Ouvrir la fiche pour consulter les informations, les documents, les contrats et les tickets liés.

Relier les informations

La force de GLPI vient des liens entre les objets. Un ticket associé à un ordinateur donne accès à l'historique de l'équipement. Un contrat associé à un actif facilite le suivi de garantie. Un document rattaché à une fiche conserve les procédures ou justificatifs utiles.

  • Associer les tickets aux actifs concernés lorsque l'information est connue.
  • Ajouter les documents utiles : captures, devis, contrats, procédures ou justificatifs.
  • Renseigner les lieux, utilisateurs et groupes pour faciliter les recherches.
  • Utiliser la base de connaissances pour transformer les résolutions récurrentes en articles durables.

Bonnes pratiques

  • Nommer les tickets avec un titre court et précis.
  • Distinguer incident, demande, problème et changement lorsque le processus le prévoit.
  • Tenir les statuts à jour pour donner une vision fiable de l'activité.
  • Eviter les fiches d'actifs incomplètes : le parc perd de la valeur si les informations sont trop pauvres.
  • Documenter la solution finale, surtout pour les incidents récurrents.
